WebGL (Web Graphics Library) est une API JavaScript qui facilite la conception de graphiques 2D et 3D directement depuis le navigateur web. Elle permet d’utiliser le standard OpenGL ES au sein d’une page web, d’un livre numérique au format ePUB 2, d’un diaporama, ou bien d’une application basée sur le standard HTML5. Pour ce faire, elle s’appuie sur le langage JavaScript, des données au format JSON, et l’accélération matérielle pour les calculs et le rendu 3D à l’aide des pilotes OpenGL ES du ou des processeurs graphiques du terminal informatique (ordinateur, smartphone, tablette numérique, etc.)

  1. Étape 1 : Fonctionnement
  2. Étape 2 : Implémentations
  3. Étape 3 : Limitations

Étape 1 : Fonctionnement

WebGL permet d’afficher, de créer et de gérer dynamiquement des éléments graphiques complexes en 3D dans la fenêtre du navigateur web d’un client. Lorsqu’un élément graphique de type WebGL est inclus dans une page web, le navigateur exécute un programme en JavaScript utilisant l’interface WebGL. La bibliothèque WebGL appelle à son tour le pilote OpenGL ES du système d’exploitation qui se chargera de faire les calculs nécessaires à l’affichage sur l’écran, en exploitant si possible l’accélération matérielle du ou des processeurs graphiques du terminal.

Étape 2 : Implémentations

WebGL est actuellement implémenté dans la plupart des grands navigateurs modernes. Par exemple, Firefox, Google Chrome, Safari, et Opera prennent en charge WebGL. Cependant, cette technologie reste assez méconnue du grand public en raison de sa relative nouveauté.

Étape 3 : Limitations

WebGL est basé sur OpenGL ES 2.0, une version d’OpenGL destinée aux systèmes embarqués. Contrairement à OpenGL 1.0, elle n’a pas les fonctions d’appel fixe. Par exemple, plutôt que de créer les objets face par face avec un appel de fonction à chaque face, WebGL utilise une approche différente.
